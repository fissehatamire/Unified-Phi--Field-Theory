🚀 Deploy to GitHub Pages — Step by Step

1. Create a new repository

Go to github.com/new and create a public repository named Unified-Phi-Theory (or any name you prefer).

Do not initialize with README, .gitignore, or license — we'll add our own files.

2. Upload the HTML file

Download the complete index.html file (the page you're reading now) and upload it to your repository.

Alternatively, use the command line:

git clone https://github.com/fissehatamire/Unified-Phi-Theory.git
cd Unified-Phi-Theory
cp /path/to/index.html ./
git add index.html
git commit -m "Add Unified Φ-Field Theory HTML"
git push origin main

3. Add a README (optional but recommended)

Create a README.md file in your repository with this content:

4. Enable GitHub Pages

  1. Go to your repository on GitHub
  2. Click Settings → Pages (in the left sidebar)
  3. Under Branch, select main and /(root)
  4. Click Save

After a few minutes, your site will be live at:

https://YOUR-USERNAME.github.io/Unified-Phi-Theory/

5. (Optional) Add a custom domain

In the Pages settings, you can add a custom domain under the Custom domain field.

📄 _config.yml (for Jekyll)

If you want to use Jekyll themes, create a _config.yml file in your repository root:

# _config.yml
title: Unified Φ-Field Theory
description: A complete synthesis of physics, consciousness, metaphysics, philosophy, and pure mathematics
theme: jekyll-theme-cayman

This is optional — the HTML works perfectly without it.

🔧 Troubleshooting

  • MathJax not loading? — Check your internet connection; MathJax loads from CDN.
  • Page not updating? — GitHub Pages may take 1–2 minutes to deploy. Clear your browser cache.
  • 404 error? — Ensure your repository is public and the branch is set to main in Pages settings.
